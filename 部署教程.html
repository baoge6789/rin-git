🚀 Rin 博客 · 部署教程

一、安装 Bun

# Windows PowerShell（管理员身份运行） powershell -c "irm bun.sh/install.ps1 | iex" # 验证安装 bun --version

二、克隆项目

# 进入工作目录 cd /d D:\记事本 # 克隆 Rin 项目 git clone https://github.com/openRin/Rin.git # 进入项目目录 cd Rin

三、安装依赖

# 使用国内镜像源安装 bun install --registry https://registry.npmmirror.com

四、配置环境变量

# 复制示例配置 copy .env.example .env.local # 编辑配置 notepad .env.local

⚠️ 关键规则 以下三个位置的名称必须完全一致：

  • .env.local 中的 DB_NAME → rin
  • Cloudflare D1 数据库的名称 → rin
  • Worker 绑定的变量名称 → DB

.env.local 完整配置-自己修改（-***必改）

# 站点配置 NAME=Rin DESCRIPTION=My Personal Blog AVATAR=https://example.com/avatar.png PAGE_SIZE=5 RSS_ENABLE=false # Cloudflare 凭证-*** CLOUDFLARE_ACCOUNT_ID=你的账户ID CLOUDFLARE_API_TOKEN=你的API令牌 # 管理员账号-*** ADMIN_USERNAME=admin ADMIN_PASSWORD=你的密码 # JWT 密钥（随机字符串-***） JWT_SECRET=RinLocalJWTSecret2026!@#SecureKey # 数据库名称 DB_NAME=rin # S3/R2（本地留空） S3_ENDPOINT= S3_BUCKET= S3_ACCESS_KEY_ID= S3_SECRET_ACCESS_KEY= # GitHub OAuth（留空） RIN_GITHUB_CLIENT_ID= RIN_GITHUB_CLIENT_SECRET=

五、部署

# 一键部署 bun run deploy

💡 说明 bun run deploy 自动完成：构建前端 → 构建后端 → 创建 D1 数据库 → 运行迁移 → 部署到 CF。

六、部署域名

🌐 访问地址

  • Pages 默认域名：https://rin-client.pages.dev
  • 自定义域名：https://----------/

七、常用命令

🚀 完整部署 bun run deploy
📦 构建前端 bun run build:client
⚙️ 构建后端 bun run build:server
💻 本地开发 bun run dev
🧹 清理缓存 bun pm cache rm
📦 安装依赖 bun install --registry https://registry.npmmirror.com

八、本地文件夹说明

📁 client/ 前端源代码
📁 server/ 后端源代码
📁 node_modules/ 项目依赖（约 1GB）
📁 dist/ 构建产物（可删除）
📁 .wrangler/ 本地模拟数据（可删除）
📄 .env.local 环境变量配置

九、安全提醒

  • 🔐 .env.local 含 Cloudflare API 令牌，切勿公开
  • ☁️ API 令牌权限：D1 · Pages · Workers · R2 编辑
